# Frontend Checklist

이 문서는 프론트엔드 변경 후 PR 전에 빠르게 점검할 항목을 정리합니다.

구조 원칙과 설계 배경은 [FRONTEND.md](../FRONTEND.md)를 기준으로 확인합니다.

## Structure

- 페이지가 조합 역할을 넘어서 도메인 상태/행동까지 직접 들고 있지 않은가
- 공용 레이아웃에서 조합한 공유 상태를 소비처가 다시 직접 조합하고 있지 않은가
- 도메인 전용 UI와 공용 UI 경계가 흐려지지 않았는가

## State

- 같은 도메인 규칙 계산이 여러 파일에 중복되지 않는가
- store 원본 상태와 view 파생 상태 책임이 섞이지 않았는가
- controller 훅이 필요한 상호작용을 컴포넌트 본문에 그대로 두고 있지 않은가

## UX

- 로딩, 에러, 빈 상태가 필요한 화면에서 누락되지 않았는가
- 키보드 동작, 포커스 이동, 접근성 속성이 깨지지 않았는가
- mock 데이터나 임시 분기 처리가 남아 있지 않은가

## Cleanup

- 변경한 흐름에 맞게 타입 정의를 함께 갱신했는가
- 대응하는 문서와 plan 문서를 함께 갱신했는가
- 설명이 필요한 복잡한 흐름에는 `// INFO:` 주석을 남겼는가
