# Pomodoro Flow

이 문서는 Tomado의 포모도로 타이머가 사용자 관점에서 어떻게 동작해야 하는지 정리합니다.

구현 구조와 내부 책임은 [Timer Domain](../design-docs/timer.md)과 [Timer README](../../src/features/timer/README.md)를 기준으로 확인합니다.

## 목표

- 사용자가 집중 세션과 휴식 세션을 끊김 없이 반복할 수 있어야 한다.
- 현재 세션 상태를 메인 화면, 집중 모드, 상단 진행 바에서 일관되게 확인할 수 있어야 한다.
- 세션 시작, 중단, 완료 시 상태 전이가 예측 가능해야 한다.

## 핵심 사용자 흐름

### 1. 타이머 시작

1. 사용자는 메인 타이머 영역에서 재생 버튼을 누른다.
2. 현재 세션이 시작된다.
3. 남은 시간, 진행률, 세션 타입 표시가 즉시 갱신된다.
4. 상단 진행 바와 문서 제목도 같은 세션 상태를 반영한다.

### 2. 세션 진행

1. 타이머는 앱 전역에서 하나의 실행 흐름으로 동작한다.
2. 사용자가 메인 화면, 집중 모드, 상단 진행 바 중 어디를 보고 있어도 같은 남은 시간과 진행률을 본다.
3. 세션이 진행되면 남은 시간, 진행률, 세션 인디케이터가 함께 갱신된다.

### 3. 집중 모드 진입

1. 사용자는 집중 모드로 들어갈 수 있다.
2. 집중 모드에서도 동일한 타이머 상태를 이어서 본다.
3. 배경 전환, TODO 펼침/접힘, 방향키 제어는 집중 모드 전용 상호작용으로 동작한다.
4. 집중 모드 진입 시 종료 방법을 안내하는 토스트가 한 번 표시된다.

### 4. 같은 날 세션 복구

1. 사용자가 타이머 도중 새로고침하거나 탭을 다시 열어도, 같은 날이라면 마지막 세션 문맥을 복구한다.
2. 복구 대상은 현재 세션 타입과 세트 진행 문맥이다.
3. 남은 시간을 초 단위로 이어서 복구하지는 않고, 복구된 세션의 기본 시간으로 다시 시작한다.
4. 휴식 세션에서 나갔다면 다시 휴식 세션부터 보이도록 유지한다.

### 5. 세션 완료

1. 현재 세션 시간이 0이 되면 세션이 완료된다.
2. 완료 시 다음 세션 타입으로 자동 전환된다.
3. 집중 세션 완료 후에는 짧은 휴식 또는 긴 휴식으로 넘어간다.
4. 세션 완료 알림과 기록 처리 흐름이 이어진다.

### 6. 휴식 건너뛰기

1. 사용자는 `short_break`, `long_break` 상태에서만 휴식을 건너뛸 수 있다.
2. 휴식을 건너뛰면 다음 세션 전환 규칙을 즉시 적용한다.
3. 다음 세션은 자동 재생하지 않고 대기 상태로 준비한다.
4. 스킵된 휴식은 서버 세션 기록에서 `skipped` 상태로 구분된다.

### 7. 세션 중단

1. 사용자가 정지 버튼을 누르면 중단 확인 흐름이 열린다.
2. 중단이 확정되면 현재 세션은 기본 시간으로 복구된다.
3. 진행 중 세션 상태와 active session 정보도 함께 정리된다.

## 세션 규칙

- 세션 타입은 `focus`, `short_break`, `long_break`로 구분한다.
- 현재 세션의 기본 시간은 세션 타입에 따라 다르다.
- 집중 세션 누적 횟수에 따라 다음 휴식 타입이 바뀐다.
- 세션 인디케이터는 현재 세트 기준 집중 세션 진행 수를 보여준다.

## 표시 규칙

- 타이머 남은 시간은 `mm:ss` 형식으로 노출한다.
- 진행률은 현재 세션의 기본 시간 대비 남은 시간으로 계산한다.
- 메인 타이머, 집중 모드, 상단 진행 바, 문서 제목은 같은 세션 기준을 사용한다.
- 세션 타입에 따라 타이머 톤과 보조 색상이 달라질 수 있다.

## 예외 및 실패 처리

- 사용자가 세션 중 다른 화면으로 이동해도 타이머는 계속 진행되어야 한다.
- 집중 모드 진입/종료와 타이머 진행 상태는 분리된다.
- 세션 시작/종료 API 호출 실패 시 사용자 상태가 완전히 꼬이지 않도록 복구 가능해야 한다.
- 새로고침 또는 세션 복구 이후에도 현재 세션 문맥이 가능한 범위에서 유지되어야 한다.

## 현재 구현 메모

- 타이머 표시 상태는 `AuthLayout`에서 공통 `timerSession`으로 조합한다.
- 메인 화면, 집중 모드, 상단 진행 바, 메타데이터 갱신은 같은 `timerSession`을 사용한다.
- 타이머 실행 흐름은 전역 ticker가 담당한다.
- 집중 모드 전용 상호작용은 별도 controller 훅으로 분리되어 있다.
