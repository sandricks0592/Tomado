# BGM Player

이 문서는 Tomado의 배경음악 플레이어가 사용자 관점에서 어떻게 동작해야 하는지 정리합니다.

구현 구조와 내부 책임은 [Settings Domain](../design-docs/settings.md)를 기준으로 확인합니다.

## 목표

- 사용자가 앱 어디에서든 배경음악을 켜고 끌 수 있어야 한다.
- 트랙 선택, 이전/다음 이동, 볼륨 조절, 재생 위치 복원이 일관되게 동작해야 한다.
- 새로고침이나 화면 이동 이후에도 마지막 재생 상태를 가능한 범위에서 복구해야 한다.

## 핵심 사용자 흐름

### 1. 플레이어 열기

1. 사용자는 배경음악 진입 버튼을 통해 플레이어 UI를 연다.
2. 현재 선택된 트랙 카테고리와 재생 상태를 볼 수 있다.
3. 트랙 데이터가 준비되면 카테고리 카드 목록이 활성화된다.

### 2. 재생 / 일시정지

1. 사용자는 재생 버튼으로 현재 선택 트랙을 재생한다.
2. 다시 누르면 일시정지된다.
3. 선택된 트랙이 없으면 기본 트랙부터 재생이 시작된다.

### 3. 트랙 이동

1. 사용자는 이전/다음 버튼으로 트랙을 이동할 수 있다.
2. 카테고리 카드를 직접 눌러 해당 카테고리 트랙으로 바로 전환할 수 있다.
3. 트랙이 바뀌면 재생 위치는 0부터 다시 시작한다.

### 4. 볼륨 조절

1. 사용자는 range input으로 볼륨을 조절한다.
2. UI의 값 변경은 실제 오디오 볼륨에 바로 반영된다.

### 5. 자동 재생 및 복구

1. 사용자가 이전에 재생 중이던 상태였다면 새로고침 후에도 재생 상태를 최대한 복구한다.
2. 브라우저 autoplay 정책 때문에 즉시 재생이 막히면, 첫 사용자 인터랙션 시 재시도를 준비한다.
3. 현재 트랙, 볼륨, 재생 여부, 재생 위치는 저장된 상태를 기준으로 복원한다.

### 6. 트랙 종료

1. 현재 트랙이 끝나면 다음 트랙으로 자동 이동한다.
2. 다음 트랙이 없으면 재생 상태가 종료된다.

## 표시 규칙

- 현재 선택된 트랙 카테고리는 플레이어 카드에서 active 상태로 표시된다.
- 플레이어는 현재 트랙, 재생 여부, 볼륨을 즉시 반영해야 한다.
- 트랙 목록이 아직 준비되지 않았을 때는 조작이 제한될 수 있다.

## 예외 및 실패 처리

- 트랙 목록 로딩 실패 시 플레이어가 완전히 깨지지 않아야 한다.
- autoplay가 차단되더라도 사용자의 명시적 인터랙션 후 재생을 다시 시도할 수 있어야 한다.
- 저장된 트랙이 현재 목록에 없으면 사용 가능한 기본 트랙으로 fallback 해야 한다.

## 현재 구현 메모

- 현재 BGM 플레이어는 전역 `Audio` 싱글턴 기반으로 동작한다.
- `useBgmPlayer`가 UI용 상태와 핸들러를 제공한다.
- persisted 상태를 먼저 읽고 store와 audio 런타임이 같은 초기값에서 시작하도록 맞춘다.
