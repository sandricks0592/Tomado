# Daily Log Flow

## 목표

- 날짜별로 데일리로그를 작성하고, 저장 상태와 선택 상태를 명확하게 보여줍니다.

## 핵심 사용자 흐름

1. 사용자는 좌측 목록에서 로그를 고르거나 날짜를 선택합니다.
2. 선택한 날짜에 로그가 있으면 우측 편집 영역에 제목과 본문이 채워집니다.
3. 본문을 수정하면 잠시 후 자동 저장되거나, 사용자가 저장 버튼으로 직접 저장할 수 있습니다.
4. 삭제 시 즉시 사라지며, 짧은 시간 동안 토스트에서 취소할 수 있습니다.

## 현재 구현 메모

- 생성/수정/삭제 API는 연결되어 있습니다.
- 목록과 검색은 아직 테스트용 로컬 데이터 기반입니다.
- 선택된 로그는 카드 강조 상태로 표시됩니다.
- autosave 상태는 `작성중 / 저장중 / 마지막 저장` 텍스트로 노출됩니다.

## 예외 및 실패 처리

- 삭제 실패 시 에러 토스트를 보여줍니다.
- 저장 실패 처리 기준은 아직 더 보강이 필요합니다.
- 미래 날짜는 선택할 수 없습니다.
