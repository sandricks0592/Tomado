# Quality Score

도메인별 현재 품질 상태와 개선 우선순위를 기록합니다.

점수는 절대 기준이 아니라, 현재 구조 안정성 / 문서화 수준 / 남은 리팩터링 규모를 빠르게 파악하기 위한 상대 지표입니다.

## Scale

- 5: 구조와 문서가 안정적이고, 당장 큰 리팩터링 필요가 적음
- 4: 대체로 안정적이지만 일부 정리 포인트가 남아 있음
- 3: 기능은 동작하지만 구조 개선 여지가 뚜렷함
- 2: 책임 분리나 문서화가 부족해 작업 시 주의 필요
- 1: 구조와 문서가 모두 불안정한 상태

## Domain Scores

### Timer — 4.5 / 5

- store / view / controller / focus controller 역할이 비교적 명확하게 분리됨
- `AuthLayout` 기준 공통 `timerSession` 주입 구조가 정리됨
- 관련 문서가 최신화돼 있음
- 남은 작업은 큰 구조 변경보다 미세 조정 수준

### Auth — 4 / 5

- 세션 복구, 로그인/회원가입 폼, 라우트 가드 흐름이 명확함
- 쿠키 세션 기반 전략이 문서화돼 있음
- 현재 구조는 안정적이지만 도메인 상세 문서는 더 확장 가능

### Settings — 3.5 / 5

- BGM 중심 구조는 비교적 명확함
- feature README가 상세하게 정리돼 있음
- 다만 향후 타이머 설정/기타 전역 설정이 들어오면 도메인 경계 재정리가 필요할 수 있음

### Todo — 3 / 5

- store / list hook / panel / item 분리는 되어 있음
- 하지만 `TodoPanel` 책임이 크고 날짜 이동, DnD, 입력, 모달 흐름이 한곳에 몰려 있음
- 날짜별 order 규칙은 문서화와 유틸 정리가 더 필요함

### Stats — 2 / 5

- 문서화가 아직 부족함
- 실제 API 연동 전 placeholder 성격 코드가 남아 있음
- 구조와 품질 상태를 추후 다시 평가해야 함

### Log — 2 / 5

- 문서화가 아직 부족함
- 현재 도메인 구조 설명과 품질 기준이 정리되지 않음

## Current Priorities

1. Todo 구조 정리
2. Log / Stats 도메인 문서 보강
3. Settings 확장 시 도메인 경계 재확인

## Notes

- 이 문서는 리팩터링이나 큰 구조 변경 이후 함께 갱신합니다.
- 점수보다 근거와 우선순위가 더 중요합니다.
