# DailyLog API 연동

## 목표

- DailyLog 페이지를 테스트용 로컬 데이터 기반에서 실제 API 기반으로 전환한다.
- 자동저장, 삭제, 목록 조회, 검색 흐름을 실제 데이터 기준으로 정리한다.

## 범위

- `src/pages/DailyLog.tsx`
- `src/api/generated/daily-logs/daily-logs.ts`

## 현재 상태

- 생성/수정/삭제 mutation은 이미 연결되어 있다.
- 페이지 내부에 테스트용 로그 배열과 임시 조회/검색 흐름이 남아 있다.
- 삭제는 지연 삭제 + 토스트 undo 패턴으로 동작한다.
- 선택된 로그는 카드 `selected` 상태로 시각적으로 표시된다.

## 작업 단계

1. 날짜별 목록 조회 API 연결
2. 검색 흐름을 실제 API 또는 목록 기반 필터링 중 하나로 확정
3. 생성/수정/삭제 후 목록과 선택 상태 재동기화 기준 정리
4. 테스트용 로그 데이터를 제거
5. 관련 product spec / design doc 갱신

## 주요 결정 포인트

- autosave debounce 기준
- 검색을 전용 API로 처리할지, 목록 조회 기반으로 처리할지
- 생성/삭제 후 목록 재조회와 선택 상태 복구를 어떻게 처리할지

## 리스크

- 입력 중 저장 실패 처리 방식이 불분명하면 사용성 문제가 생길 수 있다.
- 목록/검색 데이터 소스가 분리되면 UI 상태 관리가 복잡해질 수 있다.

## 완료 조건

- DailyLog의 주요 조회/편집 흐름이 실제 API와 연결된다.
- 테스트용 로컬 로그 데이터가 제거된다.
- 저장 및 삭제 실패 처리와 선택 상태 동기화 기준이 정리된다.
